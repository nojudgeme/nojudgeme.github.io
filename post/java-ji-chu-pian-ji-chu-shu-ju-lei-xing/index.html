基础数据类型

char Character
byte Byte
short Short
int Integer
long Long
float Float
double Double

要了解各个基础数据类型的区别，需要简单了解计算机存储单元。

计算机存储单元

计算机系统数据只用0和1这种表现形式，一个0或者1占一个“位”，而系统中规定8个位为一个字节。电脑的各种存储器的最小的存储单位是比特，也就是位（bit，简称b），它表示一个二进制位。比位大的单位是字节（byte，简称B），它等于8个二进制位：1byte=8bit。它的二进制表示范围从00000000 ~ 11111111，换算成十进制是0 ~ 255，换算成十六进制是00 ~ ff。

因为在存储器中含有大量的存储单元，每个存储单元可以存放8个二进制位，所以存储器的容量是以字节为基本单位的。

每个英文字母要占一个字节(1byte)，一个汉字要占两个字节(2byte)。

其它常用的单位还有千字节（Kilobyte，简称K或者KB，1KB等于1024B）、兆字节（Megabyte，简称M或者MB，1MB等于1024KB）和吉字节（Gigabyte，简称G或者GB，1GB等于1024MB）。B = byte

JDK中基础数据类型的最大值/最小值,比特位

查看基础数据类型的最大值/最小值，比特位，可以通过JDK提供的包装类：MAX_VALUE、MIN_VALUE、SIZE属性查看。

以int类型为例：

public final class Integer extends Number implements Comparable<Integer> {
    /**
     * A constant holding the minimum value an {@code int} can
     * have, -2<sup>31</sup>.
     */
    @Native public static final int   MIN_VALUE = 0x80000000;

    /**
     * A constant holding the maximum value an {@code int} can
     * have, 2<sup>31</sup>-1.
     */
    @Native public static final int   MAX_VALUE = 0x7fffffff;
    
    /**
     * The number of bits used to represent an {@code int} value in two's
     * complement binary form.
     *
     * @since 1.5
     */
    @Native public static final int SIZE = 32;

    /**
     * The number of bytes used to represent an {@code int} value in two's
     * complement binary form.
     *
     * @since 1.8
     */
    public static final int BYTES = SIZE / Byte.SIZE;
}
